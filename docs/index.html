C-SPAN Washington Journal — Caller Gender Analysis

Do female and male callers speak differently? This project scrapes machine-generated transcripts of every Washington Journal episode, extracts individual caller turns, and measures differences in word length, sentence structure, hedging language, question-asking behavior, vocabulary, and sentence-level sentiment across gender and party affiliation.

The Show

Washington Journal is a live, three-hour call-in program broadcast on C-SPAN that has aired daily since January 4, 1995. Each morning, callers dial in on three separate phone lines designated by self-reported party affiliation — Republican, Democrat, and Independent — and speak directly and unscripted with the on-air host about the day’s news and politics. The host introduces the caller (often including their name and location), listens to their comment or question without interruption, and responds briefly before taking the next call. The show averages roughly 15–20 calls per hour. Because each caller has an uninterrupted speaking turn of variable length, and callers self-select into party lines, Washington Journal is one of the most accessible long-running corpora of spontaneous American political speech available for research.

Data Sources

Transcripts are collected from two sources. The primary source is fight.fudgie.org, a third-party public archive of machine-generated transcripts for C-SPAN broadcasts, produced by automated speech-to-text and covering over 10,000 episodes. The fudgie.org HTML stores per-line speaker attribution via data-speaker-name attributes in the attributed format (episodes prior to 2026) and uses a CSS class (new-speaker-cell) to mark speaker changes in the unattributed format (2026+ episodes). A supplementary scraper extracts closed captions from C-SPAN’s official YouTube channel using yt-dlp. The current dataset covers — episodes and — caller turns (— gender-labeled).

Corpus Construction

Caller turns are identified by one of two methods depending on the fudgie.org episode format. In the attributed format (pre-2026 episodes), each transcript line carries a speaker name. Lines attributed to speakers matching the pattern [firstname] in [state] or labeled “Unidentified” are treated as caller turns; host turns are identified by a lookup table of known Washington Journal hosts. In the unattributed format (2026+ episodes), no speaker names are present. Caller turns are identified by the host’s party-line introduction phrase — e.g., “Republican line, go ahead” or “Democrat line, good morning” — and all turns between two such introductions are concatenated to form the caller’s text. Turns shorter than 15 words are excluded to remove off-air fragments; turns longer than 600 words are excluded to remove transcript artifacts and advertisements. Each caller turn is treated as one independent observation. Duplicate rows arising from the same episode appearing at multiple URL fragments are removed by normalizing the episode identifier and deduplicating on (episode ID, caller text).

Metadata per Caller Turn

For each extracted turn the following fields are recorded:

  • Caller name — First name extracted from the host’s spoken introduction using regular expressions (e.g., “Republican line, John from Texas” → “John”). Absent when the host does not name the caller.
  • Party line — Republican, Democrat, or Independent, parsed directly from the host’s introduction phrase.
  • Host name & gender — Identified by matching known Washington Journal host names against the full episode transcript; host gender is drawn from a manually curated lookup table.
  • Caller city & state — Parsed from introduction phrases such as “from Portland, Oregon” or “Roy in North Dakota” using a regex against a list of all 50 US state names. Only available when the host mentions a location.
  • Call hour (Eastern Time) — Derived from the episode broadcast start hour (encoded in the episode identifier string) plus the caller’s audio offset in seconds, extracted from play_ep() JavaScript calls embedded in the fudgie.org HTML, divided by 3600.
  • Call duration (seconds) — The caller’s last transcript timestamp minus their first, from the same play_ep() timestamps.
  • Day of week — Derived from the broadcast date in the episode identifier (YYYYMMDD prefix).

Gender Inference

Caller gender is inferred automatically in order of precedence. First, if the host names the caller, the first name is queried against the gender-guesser Python library, which classifies names as male, mostly_male, female, mostly_female, or andy (androgynous / ambiguous) based on a large international name database. Names classified as male or mostly_male are labeled male; female or mostly_female are labeled female; andy, absent, or unrecognized names yield unknown. Second, when name inference fails, the caller’s transcript text is scanned for gendered salutations: sir or gentleman indicate male; ma’am or madam indicate female. Turns where neither signal resolves a gender are labeled unknown and excluded from gender-comparative analyses. No manual labeling was performed. The automated approach systematically underpredicts labeling rates: gender-neutral names, non-English names, and calls where the host does not introduce the caller by name all produce unknown labels.

Linguistic Features

All features are computed from the raw transcript text of the caller’s turn using regular expressions and word-list lookups; no external NLP models are used for feature extraction (sentiment is a separate step, below).

  • Word count — Total whitespace-delimited tokens in the caller’s turn.
  • Sentence count — Number of segments produced by splitting on sentence-ending punctuation (. ! ?). Empty segments are discarded.
  • Question count / Question ratio — A sentence is classified as a question if it ends with “?” or begins with one of the following interrogative or auxiliary-inversion words: do, does, did, is, are, was, were, will, would, can, could, have, has, why, what, how, when, where, who, which, whose, wouldn’t, don’t, isn’t, aren’t, can’t, didn’t. Question ratio is question count ÷ sentence count.
  • Avg words per sentence — Word count ÷ sentence count. A proxy for syntactic complexity.
  • Vocabulary diversity (type–token ratio) — Number of distinct lowercased word types ÷ total word tokens. Ranges from near 0 (maximal repetition) to 1.0 (no word used more than once). Equivalent to the type–token ratio (TTR).
  • Hedging rate — Count of hedging phrases ÷ word count. Hedging signals epistemic uncertainty or tentativeness. Phrases counted: I think, I feel, I believe, I guess, maybe, perhaps, possibly, it seems, sort of, kind of, I was wondering, I’m not sure, I don’t know, might be, could be. Matches are case-insensitive; multi-word phrases are matched as contiguous substrings.
  • Modal verb rate — Count of modal verbs (would, could, should, might, may, ought) ÷ word count. Modal verbs express conditionality, possibility, or obligation and are commonly used to soften assertions.
  • First-person pronoun rate — Count of I, me, my, mine, myself ÷ word count. A measure of self-referential speech; higher values indicate callers foregrounding their own experience or perspective.
  • Second-person pronoun rate — Count of you, your, yours, yourself ÷ word count. A measure of direct address to the host or a general audience.
  • First meaningful word — The first word of the caller’s turn after stripping filler tokens (um, uh, well, yes, yeah, no, so, and, but, i, you, know, like, okay, ok, hi, hello, good). Reveals whether callers open with a declarative concept, a question word, or a named entity.

Sentiment Analysis

Sentence-level sentiment is scored with VADER (Valence Aware Dictionary and sEntiment Reasoner; Hutto & Gilbert, 2014), a rule-based lexicon model designed for informal and spoken language. VADER assigns each sentence a compound score in [−1, +1], constructed by summing valence scores from a sentiment lexicon and applying rules for negation (“not good”), intensifiers (“very”), capitalization emphasis, and punctuation. Each caller turn is split into sentences; per-sentence compound scores are averaged to produce a per-turn mean compound score. Three additional fractions are recorded: the share of sentences with compound > 0.05 (% positive), compound < −0.05 (% negative), and within [−0.05, 0.05] (% neutral).

Host Responsiveness

For each caller turn, the host’s first spoken turn immediately following the caller is captured as host_response_text. Four metrics are derived:

  • Host response length — Word count of the host’s response.
  • Topic overlap (Jaccard similarity) — The Jaccard index between the set of non-stop-words in the caller’s text and those in the host’s response: |caller ∩ host| ÷ |caller ∪ host|. Ranges from 0 (no shared vocabulary) to 1 (identical vocabulary). Higher values indicate the host directly addressed the caller’s specific topics.
  • Compliment detection — A Boolean flag: True when the host’s response contains a phrase from a predefined list including good question, great point, excellent point, very interesting, well said.
  • Follow-up question — A Boolean flag: True when the host’s response contains a question mark, indicating active engagement rather than a simple acknowledgment.

Host response data is only available for episodes scraped after this feature was added to the scraper; earlier rows in the dataset have an empty host_response_text field. In the unattributed format (2026+), the host response is approximated as turns following the caller’s last long speech segment, which is a heuristic.

Statistical Methods

Group differences between female and male callers on all continuous linguistic features are tested with the Mann–Whitney U test (implemented in SciPy), a non-parametric rank-sum test that makes no assumption of normality and is appropriate for the skewed, bounded distributions typical of linguistic rate measures. Tests are two-tailed. Significance thresholds: p < 0.001 (***), p < 0.01 (**), p < 0.05 (*); p ≥ 0.05 is labeled not significant (ns). Error bars on bar charts show ±1 standard error of the mean (SE).

Visualization & Tools

Interactive charts are rendered with Plotly.js (all violin, bar, scatter, histogram, and time-series plots; supports pan, zoom, and PNG/SVG export). Sortable, filterable data tables are built with Tabulator. The geographic bubble map uses Leaflet.js with CartoDB light tile layers. All analysis code is in Python using pandas and SciPy.

Limitations

Machine-generated transcripts contain systematic errors, particularly for proper nouns, names, overlapping speech, and accented speakers. Gender inference is approximate and produces high unknown rates (>90% in 2026+ unattributed episodes) because many hosts do not introduce callers by name. Call duration, call hour, and host response text are only available for episodes scraped with the current scraper version; earlier rows have these fields empty. Geographic state data is only available when the host verbally mentions the caller’s location. The dataset covers a sample of episodes and grows incrementally — run make fudgie-big to append additional episodes.

Do You Have a Question?

How often do callers ask questions versus making statements? Does gender, party affiliation, or the host’s gender shape that behaviour? How does tone — positive, negative, neutral — vary across groups?

Questions vs Statements — by Gender

Question ratio and key speech metrics, female vs male callers (gender-labeled turns only).

Question ratio — The fraction of sentences classified as questions: Qratio = Qcount / Scount, where Qcount is the number of question sentences and Scount is total sentence count. Ranges from 0 (no questions) to 1 (every sentence is a question).

How a question is identified — A sentence is classified as a question if (a) it ends with “?”, or (b) its first word is one of: do, does, did, is, are, was, were, will, would, can, could, have, has, why, what, how, when, where, who, which, whose, wouldn’t, don’t, isn’t, aren’t, can’t, didn’t. This heuristic captures both direct questions and auxiliary-inversion questions.

Vocab diversity (type–token ratio, TTR) — TTR = |V| / N, where |V| is the number of distinct word types (lowercased) and N is the total number of tokens. TTR near 1.0 indicates near-zero repetition; lower values indicate more repeated vocabulary. TTR is sensitive to text length and should be compared within similar word-count ranges (Templin, 1957).

1st-person rate — Count of {I, me, my, mine, myself} ÷ N. Measures self-reference; higher values indicate the caller foregrounds their own experience or identity.

2nd-person rate — Count of {you, your, yours, yourself} ÷ N. Measures direct address to the host or a generalised “you” audience.

Error bars — Each bar shows the group mean ± 1 standard error of the mean (SE = σ / √n). SE quantifies the precision of the mean estimate, not the spread of individual observations.

Tone of the Comment — Sentiment Analysis

Sentence-level sentiment scored with VADER (compound: −1 = most negative, +1 = most positive). Bar charts show six groups: Dem·Female, Dem·Male, Rep·Female, Rep·Male, Ind·Female, Ind·Male. Pink = female, blue = male.

VADER sentiment model — Valence Aware Dictionary and sEntiment Reasoner (Hutto & Gilbert, 2014). A rule-based lexicon model designed for informal and social-media language. For each sentence VADER computes: positive valence score p, negative valence score n, neutral valence score neu, and a compound score C = normalize(Σ valence), where the normalization uses α = 15 to bound C to [−1, +1]. VADER handles negation (“not good”), degree modifiers (“very”, “barely”), ALL-CAPS emphasis, and punctuation. Reference: Hutto, C.J. & Gilbert, E.E. (2014). ICWSM. doi:10.1609/icwsm.v8i1.14550

Per-turn compound score — Each caller turn is split into sentences on terminal punctuation; VADER scores each sentence; the mean of sentence-level compound scores is the per-turn score. Sentences with |C| ≤ 0.05 are neutral; C > 0.05 is positive; C < −0.05 is negative.

% positive / negative / neutral sentences — The fraction of sentences in a turn falling into each sentiment category. These three fractions sum to 1 for each turn.

Scatter: % positive sentences vs call duration — Only visible when call-duration data is available (requires re-scraping with the updated scraper). X-axis is the fraction of positive sentences; Y-axis is call duration in seconds from play_ep() timestamps.

How Do They Say It?

Linguistic style beyond simple question-asking: hedging language, modal verb use, pronoun patterns, first-word choices, vocabulary scatter, and — once enough re-scraped episodes accumulate — how the host responds.

Hedging, Modal Verbs & Pronoun Use

Hedging rate — Count of hedging phrases ÷ N. Hedging language signals epistemic uncertainty or face-saving tentativeness (Hyland, 1996. Written Communication, 13(2), 251–281. doi:10.1177/0741088396013002004). Phrases counted: I think, I feel, I believe, I guess, maybe, perhaps, possibly, it seems, sort of, kind of, I was wondering, I’m not sure, I don’t know, might be, could be. All matches are case-insensitive continuous substrings.

Modal verb rate — Count of {would, could, should, might, may, ought} ÷ N. Modal verbs express epistemic possibility, deontic obligation, or conditionality and are frequently used to soften assertions or frame hypothetical situations.

1st- and 2nd-person pronoun rates — See definitions in the “Questions vs Statements” section above. All rates are divided by N (total word count) so that longer turns do not inflate absolute counts.

First Meaningful Word

First meaningful word — The first word of each caller’s turn after removing the following filler tokens: um, uh, well, yes, yeah, no, so, and, but, i, you, know, like, okay, ok, hi, hello, good. The opener reveals whether a caller leads with a declarative concept, a question word, or a named entity — a proxy for call intent (statement vs. question) that does not require sentence boundary detection.

How does the host respond after each caller? Metrics are computed from the host’s first turn immediately following the caller. Only available for episodes scraped after this feature was added. — calls have host response data.

Host response length — Word count of the host’s first spoken turn immediately after the caller finishes. Longer responses may indicate the host found the call more substantive.

Topic overlap (Jaccard similarity) — J(A,B) = |A ∩ B| / |A ∪ B|, where A is the set of non-stop-words in the caller’s text and B is the set of non-stop-words in the host’s response. J = 0 means no shared vocabulary; J = 1 means identical vocabulary. Higher values indicate the host directly addressed the caller’s specific topics. Stop-words (function words, pronouns, common auxiliaries) are excluded to focus on content words.

Compliment detection — Boolean flag: True when the host’s response matches any phrase in the set {good question, great question, excellent question, good point, great point, very interesting, interesting point, well said}, case-insensitive. Indicates explicit praise from the host.
